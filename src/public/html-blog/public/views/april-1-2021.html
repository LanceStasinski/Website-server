Learning HTML and CSS

I took a web programming class in high school, which was about seven or eight years ago. Surprisingly, some of the concepts of HTML and CSS are coming back to me pretty quickly. It also helps that I've been troubleshooting with R for the past year and a half while trying to run statistical analyses using multivariate machine learning models. Overall, HTML and CSS has been pretty fun to learn, and I appreciated the enthusiam about coding from Udacity's Daniel Silver Baker.

What I like about Udacity

Udacity does a great job with introducing students to outside resources like MDN Web Docs. Being confortable with engaging with other resources has already proved useful for constructing this website. I also think Udacity, and especially Daniel Silver Baker, do a great job at making coding accessible to a broad audience. Coding can come off as daunting to a lot of people, so Udacity's welcoming attitude makes coding seem more fun and exciting. I've also enjoyed the short quizes along the way and the hands-on practice.

What could use some improvement

Although I like the hands-on practice, I think there could be substantially more. I have been utilizing freecodecamp's tutorials as a secondary source for learning HTML and CSS as I feel the extra practice is needed. It's not neccessarily a bad thing that Udacity does not hold your hand the whole way, but it does feel like they could offer more considering it is a paid service. However, this is my first impression and there is still much I have yet to experience with Udacity.
